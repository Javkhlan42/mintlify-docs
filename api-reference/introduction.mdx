## Ерөнхий мэдээлэл

Yellow Book API нь байгууллагын мэдээллийг хайх, дэлгэрэнгүй харах, холбоо барих болон webhook-оор шинэчлэл хүлээн авах RESTful API юм.

## Үндсэн боломжууд

- Байгууллага хайх (нэр, түлхүүр үг, дүүрэг, ангилал)
- Байгууллагын дэлгэрэнгүй мэдээлэл авах
- Холбоо барих утас, и-мэйл авах
- Байгууллагын өөрчлөлтийн webhook (админ)

## API Base URL

Бүх API хүсэлтүүд дараах суурь URL-аас эхэлнэ:

```
https://api.yellowbook.mn/v1
```

## Баталгаажуулалт (Authentication)

Ихэнх endpoint-ууд нээлттэй (public) боловч заримад нь API key эсвэл Bearer token шаардлагатай байж болно.

### Жишээ:

**Public endpoint:**
```bash
curl -X GET "https://api.yellowbook.mn/v1/organizations/search?query=кофе" 
```

**Token шаардлагатай endpoint:**
```bash
curl -X GET "https://api.yellowbook.mn/v1/organizations/123" \
  -H "Authorization: Bearer <access_token>"
```

## Стандарт хариултын бүтэц

Бүх API endpoint-үүд стандарт бүтэцтэй JSON хариулт буцаана:
```json
{
  "value": <Үндсэн өгөгдөл>,
  "data": <Нэмэлт өгөгдөл эсвэл null>,
  "msgList": [<мэдэгдэл>],
  "attrs": {<нэмэлт атрибут>},
  "status": "Success" | "Error"
}
```

## Алдааны кодууд

| HTTP Code | Тайлбар                | Шийдэл                        |
|-----------|------------------------|-------------------------------|
| 200       | Амжилттай              | -                             |
| 400       | Буруу хүсэлт           | Параметр, body шалгах         |
| 401       | Баталгаажуулалт амжилтгүй | Token, эрх шалгах         |
| 403       | Хандах эрхгүй          | Эрхийн тохиргоо шалгах        |
| 404       | Өгөгдөл олдсонгүй      | ID, параметр зөв эсэхийг шалгах |
| 429       | Rate limit хэтэрсэн    | Түр хүлээгээд дахин оролдох   |
| 500       | Серверийн алдаа        | Дахин оролдох, support-той холбогдох |

### Алдааны хариултын жишээ
```json
{
  "value": null,
  "data": null,
  "msgList": ["Байгууллага олдсонгүй"],
  "attrs": {},
  "status": "Error"
}
```

## Rate Limiting

API хүсэлтүүд rate limit-тэй:
- Стандарт лимит: 100 хүсэлт/минут, endpoint бүрээр
- Burst лимит: 20 хүсэлт/секунд
Лимит хэтэрвэл HTTP 429 статус код буцаана.
